# kebab-case File & Directory Naming

**Rule:** All file and directory names MUST use **kebab-case** — lowercase ASCII letters and digits separated by single hyphens (`-`).

**Pattern:** `^[a-z0-9]+(-[a-z0-9]+)*(\.[a-z0-9]+)*$`

- Correct: `user-profile.ts`, `auth-guard.test.ts`, `http-client/`
- Incorrect: `UserProfile.ts`, `userProfile.ts`, `user_profile.ts`

**Notes:**

- Dots are reserved for compound extensions (`.test.ts`, `.types.ts`, `.d.ts`), not as word separators.
- File names are independent of identifier casing inside the file (e.g., a React component file `user-card.tsx` may export `UserCard`).
- Ecosystem-mandated names are exempt: `README.md`, `LICENSE`, `Dockerfile`, `tsconfig.json`, framework-reserved files (e.g., `[id].tsx`, `_app.tsx`).

**Enforcement:** Enable `unicorn/filename-case` (`{ case: 'kebabCase' }`) in ESLint; CI MUST fail on violations.
