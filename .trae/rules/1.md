# Strict TypeScript Typing & Runtime Validation

**Scope:** All TypeScript code in this project.

**Rule:** Never escape the type system. Rely on strict static types, and validate all external input at runtime with `zod`.

**Prohibited:**

- `any` (explicit or implicit), `Function`, `Object`, `{}` as types — use `unknown` and narrow it.
- Type assertions used to silence errors: `as T`, `as unknown as T`, non-null `!`.
- Suppression directives: `@ts-ignore`, `@ts-expect-error`, `@ts-nocheck`.
- Disabling `strict` or any of its sub-flags in `tsconfig.json`.

**Required:**

1. `tsconfig.json` MUST set `"strict": true`, `"exactOptionalPropertyTypes": true`.
2. Model domains precisely: discriminated unions, `readonly`, branded primitives, exhaustive `never` checks.
3. Every value crossing a trust boundary (HTTP payloads, query params, env vars, third-party responses, file/IPC/queue data) MUST be parsed through a `zod` schema before use.
4. Derive static types via `z.infer<typeof Schema>` — one schema is the single source of truth; do not declare parallel `interface`/`type`.
5. Use `schema.parse()` to fail fast at boundaries; reserve `safeParse()` for contracts that explicitly return typed errors. Never silently coerce invalid input.
6. Narrow `unknown` only via schema validation or user-defined type guards (`x is T`).

**Enforcement:**

- ESLint MUST enable `@typescript-eslint/no-explicit-any`, `no-unsafe-*`, `no-non-null-assertion`, `ban-ts-comment`, `consistent-type-assertions`.
- CI MUST run `tsc --noEmit` and `eslint --max-warnings 0`; violations block merge.
- Reviewers MUST reject PRs introducing assertions, suppressions, or unvalidated input — regardless of urgency.

**Rationale:** Escape hatches convert compile-time guarantees into latent runtime defects. Strict typing plus `zod` ensures the contract holds identically at both layers.
