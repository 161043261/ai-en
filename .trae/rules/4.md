# English-Only Codebase

**Rule:** This is an English-language project. All code-adjacent text MUST be written in clear, professional English. Non-English natural language (especially Chinese characters) is prohibited.

**Applies to:** identifiers, comments, log/error messages, test descriptions, commit messages, branch names, PR titles/descriptions, and in-repo documentation.

**Prohibited:**

- CJK characters anywhere in source files.
- Pinyin transliterations as identifiers (e.g., `yonghu`, `dingdan`) — use proper English terms (`user`, `order`).
- Mixed-language identifiers or comments (e.g., `// 处理 user input`).

**Permitted exceptions:**

- User-facing i18n resources (`locales/zh-CN.json`, message catalogs) — data, not code.
- Test fixtures intentionally exercising Unicode/CJK handling.
- Third-party vendored content.

**Quality bar:** Correct grammar, domain-appropriate terminology, descriptive names (`pendingInvoiceCount`, not `pic`). No slang; abbreviate only well-known terms (`id`, `url`, `api`).

**Enforcement:** ESLint MUST block CJK characters in identifiers and string literals (with an allowlist for i18n paths); CI MUST fail on violations.
